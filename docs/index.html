elegram-api

Elegram-api is a wrapper over the telegram bots api, but in the style of EO. More object-oriented, more maintainable.

Check us out at Github.

How was the idea for this project born?

I was puzzled after trying out the libs for telegram bots because I'd never written such bad code before. Libraries are forcing us to write this garbage. This is how the idea of elegram-api came about.

Here some problems that I found in other projects:

  • Empty lines
  • Compound names
  • Incorrect formatting
  • Mutability
  • Getters and Setters
  • Static methods
  • Pull request without a unit-tests
